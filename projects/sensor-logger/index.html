Overview

I want to build something real with a microcontroller before I jump into custom PCB design. The goal here is simple: read temperature, humidity, and barometric pressure from a sensor array over I²C, and log the data to an SD card with timestamps. Nothing fancy — but it forces me to actually deal with hardware communication protocols, power issues, and file systems, which is exactly the kind of friction I need right now.

The longer-term goal is to take everything I learn here — pin assignments, power regulation, decoupling, sensor wiring — and design a proper PCB that does the same job without the breadboard mess. This project is the research phase for that one.

Noisy ADC readings — tracking down the cause
⚠ Problem

The BME280 over I²C is reading fine — temperature and humidity look stable. But I added an analog soil moisture sensor on A0 and the readings are jumping around by ±80 counts even when the probe is sitting in air. That's way too much variance to be useful.

My first guess was RF interference from the USB cable. I tried shielding it — no difference. Then I noticed the 3.3V rail on my breadboard is sagging to about 3.1V under load, which could be affecting the ADC reference voltage. Still investigating. Added a 100µF cap across the power rails as a test.

Note to self Look into AREF pin — if I connect an external 3.3V reference directly, the ADC should stop being affected by rail sag. Read the datasheet section on ADC noise reduction before trying anything else.

I²C working, SD card writing, first real data
✓ Milestone

Got the full pipeline working end-to-end. BME280 is talking over I²C at address 0x76, and every 5 seconds the Arduino writes a CSV row to the SD card: timestamp, temperature, humidity, pressure. Pulled the card out and opened it in a spreadsheet — actual data. Felt good.

// Log format on SD card unsigned long ts = millis(); sprintf(buf, "%lu,%.2f,%.2f,%.2f\n", ts, bme.readTemperature(), bme.readHumidity(), bme.readPressure() / 100.0F); logFile.print(buf);

One issue: if the SD card isn't inserted on boot, the sketch hangs. I need to add a retry loop with a timeout instead of just while(!SD.begin()). Not critical but annoying.

First I²C scan — figuring out addresses
↗ Learning

Before writing any sensor code I ran the standard I²C scanner sketch to see what addresses show up on the bus. The BME280 showed up at 0x76. Good — that's the default with SDO pulled low.

Spent longer than I'd like to admit trying to figure out why my SDA/SCL pull-up resistors weren't needed — turns out the Arduino Uno's Wire library enables the internal pull-ups automatically. I had 4.7kΩ resistors wired in and everything still worked, but now I understand why the datasheet specifies external ones for longer cable runs or higher bus speeds.
