Overview

I want to build something real with microcontrollers to understand the fundamentals of communications protocols, and gain experience with reading reading datasheets before I commit to building a custom PCB. The goal for this project is simple: read temperature, humidity, , and barometric pressure from a sensor over I2C. Of course, there are many places in which I can go above and beyond what is required like storing the data locally, on a cloud server, using batteries instead of USB, etc. but for now, we'll try to get the bare minimum working.

The longer-term goal is to take everything I learn here, and design a proper PCB that does the same job without the breadboard mess. Once the PCB design is finished, I plan on designing a container for it in solidworks and have it run as device I use on the day to day in my room.

ESP32 Setup - Arduino IDE can't detect ESP32 Mar 13, 2026
⚠ Problem

Before even using the board, I first had to install some libraries in the Arduino IDE. After doing so, I set the board to "ESP32 dev module" and tried to set the port but noticed that the Port menu was greyed out.

Apparently, such a thing happens when the PC can't detect the microcontrollers. After connecting and disconnecting the ESP32, I noticed that my laptop detects the connection, but can't identify what the ESP32 is. After checking the device manager and again connecting and disconnecting multiple times I became sure that my device is not being picked up which would mean it's a driver issue.

Note to self Make sure to visit the website provided by the device manufacturer when trying to use brand new equipment. They usually contain instructions for first use and useful information (i.e. documentation for that specific model, drivers, etc.).

              // the setup function runs once when you press reset or power the board
              void setup() {
                // initialize digital pin LED_BUILTIN as an output.
                pinMode(LED_BUILTIN, OUTPUT);
              }

              // the loop function runs over and over again forever
              void loop() {
                digitalWrite(LED_BUILTIN, HIGH);  // change state of the LED by setting the pin to the HIGH voltage level
                delay(1000);                      // wait for a second
                digitalWrite(LED_BUILTIN, LOW);   // change state of the LED by setting the pin to the LOW voltage level
                delay(1000);                      // wait for a second
              }

Note that the LED_BUILTIN refers to the pin to which the builtin LED is assigned. A quick look at the documentation file tells us that LED IO2 is at pin 2.

Display - setup and testing Mar 13, 2026

The display setup was pretty straight forward. There are four pins on the display each corresponding to "GND", "VCC", "SCL", and "SDA", respectively. A quick look at the SSD1306 Datasheet tells us that for IC logic, a voltage of 1.65 to 3.3 Volts is required, which means that "VCC" must be connected to the 3.3 V pin on the ESP32. The clock and data pins of the display correspond to the I2C_SCL and I2C_SDA pins on the ESP32 (pins 22 and 21). Similarly, GND was connected to the ground pin of the ESP32.

After that I uploaded the following test code to see if the display works or not. More information on how I coded this can be found from the "example" folders of the documentation pages of the libraries used. I provided some comments to make it easier to understand. 


                #include  // required library whenever we use I2C protocol
                #include  // Adafruit graphical library used for drawing
                #include  // Specific library included when using the SSD1306 chip

                // From the examples provided in the documentation for Adafruit_SSD1306.h 
                // and by looking at the datasheet for the device we see that the following
                // variables must be defined for our specific device
                #define SCREEN_WIDTH 128 // Display width
                #define SCREEN_HEIGHT 64 // Display height
                // Skips reset (we don't have a specific pin allocated for resetting the screen so 
                // we've told Adafruit to skip hardware reset)
                #define OLED_RESET -1 

                // From the documentation, we learn that the following initializes an object called "display" 
                // with the following attributes
                Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, OLED_RESET);

                void setup() {
                  // From the documentation, we learn that display.begin() initializes the display hardware
                  // SSD1306_SWITCHCAPVCC tells the device to generate its own internal power as opposed to an external one, our device 
                  // does this so no need to change it
                  // 0x3C is the I2C address for our display, from my research, 128 x 64 resolution displays of this 
                  // brand usually have this specific address
                  // If the display fails, we display a faliure message and have an infinite loop there (error)
                  if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
                    Serial.println("SSD1306 allocation failed");
                    // infinite loop to show error
                    for(;;);
                  }

                  // clears any data from the screen buffer
                  display.clearDisplay();

                  // set text size
                  display.setTextSize(2);

                  // set text color
                  display.setTextColor(SSD1306_WHITE);

                  // begin writing at the left most part of the screen (x = 0) with 20 pixel spce from the top (y = 20)
                  display.setCursor(0, 20);

                  // print message and add line break at the end
                  display.println("Hello");
                  
                  // print message and add line break at the end
                  display.println("World!");

                  // everything above this line just loads information into the memory, this command actually displays it
                  display.display();
                }

                // there's nothing in the loop because this is a static program, we don't need anything to run many times
                void loop() {
                  // Nothing here
                }
BME280 sensor - setup and testing Mar 13, 2026
⚠ Problem

When unpacking the sensor, I noticed that the pins were separate from the sensor. At first I thought sandwitching the black block part of the pins between the sensor and the breadboard would be enough to connect it, but after running the sensor code, I noticed that nothing is being displayed. The code used to run the sensor is as follows. Sufficient comments were provided to explain functionality. 


                #include 
                #include 
                #include 
                #include  // including an additional BME280 device specific library

                #define SCREEN_WIDTH 128
                #define SCREEN_HEIGHT 64
                #define OLED_RESET -1

                Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, OLED_RESET);
                // creating the object bme which represents the sensor similar to the one above. 
                // after looking at examples provided by the documentation, I realized that it doesn't take any
                // parameters and uses the I2C default settings.
                Adafruit_BME280 bme;

                void setup() {
                  // Here, we initialize a USB serial communication between the ESP32 and our computer
                  // I noticed that the example in the documentation uses 9600 but after doing some more research,
                  // I noticed that the modern default is 115200 which is faster as well
                  Serial.begin(115200);
                  
                  // Initialize OLED
                  if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
                    Serial.println("SSD1306 allocation failed");
                    for(;;);
                  }
                  
                  // Initialize BME280
                  // This is similar to the OLED initialization above with a few small differences.
                  // the command bme.begin(0x76) initializes the sensor. the address 0x76 is the I2C address of the BME280
                  // some other devices use different addresses like 0x77
                  if (!bme.begin(0x76)) {
                    // The line below uses Serial to print an error message in the Serial Monitor of the computer within the IDE
                    Serial.println("Could not find BME280 sensor!");
                    // The following will show an error message on the OLED display
                    display.clearDisplay();
                    display.setTextSize(1);
                    display.setTextColor(SSD1306_WHITE);
                    display.setCursor(0, 0);
                    display.println("BME280 Error!");
                    display.display();
                    for(;;);
                  }

                  // prints if initialization is successfull
                  Serial.println("BME280 found!");
                }

                void loop() {
                  // Initialize variables
                  // The functions themselves are used in the examples provided by the documentation
                  // but I had to dig in the datasheet to find the format of the data that is presented which is as follows:
                  // Reads temperature in celcius
                  float temp = bme.readTemperature();
                  // reads relative humidity as a percentage
                  float humidity = bme.readHumidity();
                  // reads pressure in Pascals, division by 100.0F converts it to hecto pascals (hPa) which si a more common unit
                  // for measuring atmospheric pressure
                  float pressure = bme.readPressure() / 100.0F;
                  
                  // Display on OLED
                  display.clearDisplay();
                  display.setTextSize(1);
                  display.setTextColor(SSD1306_WHITE);
                  
                  display.setCursor(0, 0);
                  display.print("Temp: ");
                  display.print(temp);
                  display.println(" C");
                  
                  display.setCursor(0, 20);
                  display.print("Humidity: ");
                  display.print(humidity);
                  display.println(" %");
                  
                  display.setCursor(0, 40);
                  display.print("Pressure: ");
                  display.print(pressure);
                  display.println(" hPa");
                  
                  display.display();
                  
                  delay(2000);  // Update every 2 seconds
                }

BME280 setup - soldering the pins Mar 13, 2026
✓ Milestone

Thanks to amazon prime, I got the solder within a day of ordering it. I soldered the pins with the longer end of the pins sticking into the breadboard and the blocks and shorter ends sticking out. Then I sandwitched the black blocks between the breadboard and the hardware side of the sensor facing down. Since this was my first time soldering anything, I have to say that this was a pretty miserable experience. The pins were very close together and I shorted them quite a few times until the results were finally acceptable.

Note to self Be very careful when soldering sensors. If you don't work quickly enough and take too long, you might overheat the sensor which might damage some of the components.
